## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap within button |
| sm | 8px | Not used in Phase 1 |
| md | 16px | Not used in Phase 1 |
| lg | 24px | Button vertical padding (top + bottom) |
| xl | 40px | Button horizontal padding (left + right) |
| 2xl | 48px | Not used in Phase 1 |
| 3xl | 64px | Not used in Phase 1 |

Exceptions: Button touch target must be at minimum 44px tall (accessibility). With 24px vertical padding and 24px font, button height = 24 + 24 + 24 = 72px — exceeds 44px minimum. No additional exception needed.

Source: default 8-point scale; button padding derived from RESEARCH.md inline style `padding: 1rem 2.5rem` (16px / 40px) — rounded to nearest scale token.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #111111 | Overlay background, page background |
| Secondary (30%) | #1e1e1e | App body background (revealed after overlay dismissal) |
| Accent (10%) | #ffffff | Start button background, button label text |
| Destructive | not applicable | No destructive actions in Phase 1 |

Accent reserved for: Start button background fill only. All other interactive elements in future phases use a separate, phase-specific accent definition.

Color rationale:
- `#111111` is pulled directly from RESEARCH.md code example `background:#111`
- Dark theme is appropriate for an audio tool targeting video production workflows (typically dark-mode environments)
- White accent on the Start button maximizes contrast (WCAG AA ratio: #ffffff on #111111 = 18.1:1 — exceeds 4.5:1 minimum)
- `#1e1e1e` for app body provides subtle depth differentiation from the overlay without introducing a third hue

Source: RESEARCH.md inline style `background:#111` + dark-theme inference from audio/video tool context

## Interaction Contract

### Overlay Dismissal Sequence

1. Page loads → `#start-overlay` visible, `#app` hidden (`display: none`)
2. User clicks `#start-btn`
3. `ensureRunning()` called — `AudioContext` created lazily with `{ sampleRate: 44100 }`
4. If context was `suspended`: `ctx.resume()` awaited
5. Console logs: `[AudioEngine] state: running`
6. `#start-overlay` removed from DOM (`.remove()`, not hidden)
7. `#app` revealed (`style.display = ''`)

### Transition

No animated transition for Phase 1. The overlay is removed synchronously after the `await ensureRunning()` resolves. Animation is deferred — it is not a Phase 1 requirement and adds complexity to the bootstrap path.

### Focus Management

- On page load: browser default focus (no manual focus assignment needed — single interactive element)
- After overlay removal: no focus assignment needed in Phase 1 (app container has no interactive content yet)
