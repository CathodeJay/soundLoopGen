| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Button padding top/bottom (`padding: 4px 12px`) |
| sm | 8px | Row padding (`padding: 8px 0`), element gaps (`gap: 8px`) |
| md | 16px | Divider margin (`margin: 16px 0`), section padding |
| lg | 24px | App container padding (`padding: 24px`) |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: Touch target minimum height is 44px (applied to all buttons via `min-height: 44px`). This does not break the 4px scale — it is a floor constraint on interactive elements, not a spacing token.

All components are hand-rolled DOM elements (no framework). The export section appends to `#app` after the master volume section, following the exact same DOM construction pattern as `renderMixer()`.

| Element | DOM Type | Attributes / Style |
|---------|----------|--------------------|
| Second divider | `<div class="mixer-divider">` | `border-top: 1px solid #444; margin: 16px 0;` — identical to existing divider |
| Export row container | `<div>` | `display: flex; align-items: center; gap: 8px; padding: 8px 0;` |
| "Export" label | `<span>` | `font-size: 16px; font-weight: 600; color: #ffffff; min-width: 100px;` — matches "Master Volume" label |
| Duration dropdown | `<select>` | Options: "30 seconds" (value=30), "1 minute" (value=60), "2 minutes" (value=120). Default selected: 30 seconds. Style: dark bg, white text, consistent with app palette |
| Export WAV button | `<button>` | `min-height: 44px; padding: 4px 12px; border-radius: 4px; font-size: 16px; font-weight: 600;` — three visual states (see below) |

Note: The warning hint uses 14px as an exception — it is a subordinate hint below the action row, not a primary label. It also uses `#ffcc00` (amber) rather than pure white to distinguish it visually from regular labels without introducing a red "error" affordance (the situation is not an error, just a precondition).

## Interaction Contracts

### Duration dropdown
- Default value on render: `30 seconds` (value `30`)
- Persists selection for the session (no reset between exports)
- No label change needed — the dropdown reads its own selected option

### Export button
- On click (when enabled): immediately switch to `Rendering...` + disabled state; begin `OfflineAudioContext` render
- On render complete: trigger file download automatically; reset button to `Export WAV`; re-evaluate disabled state based on current sound activity
- No success toast, no success message — silent reset as specified in CONTEXT.md
- Focus ring: `outline: 2px solid #ffffff; outline-offset: 3px` on `:focus-visible` — matches existing `.btn-toggle:focus-visible` style

### Warning hint visibility
- Hint `<div>` is present in the DOM at all times after `renderExport()` runs
- Visibility is controlled by `display: block` (no sounds active) or `display: none` (at least one sound active)
- Re-evaluated on every toggle button click (same click handler that starts/stops sounds)

### `volumeMap` access
- The export function must receive `{ activeSoundIds, gains, masterGainValue }` as parameters
- `volumeMap` is local to `main.js` — it must be passed explicitly to `exportEngine.exportMix()`, not accessed by module import
- This matches the pattern described in `04-RESEARCH.md` § Pitfall 6

## Accessibility

- All interactive elements (`<button>`, `<select>`) receive `:focus-visible` outline: `2px solid #ffffff; outline-offset: 3px`
- Export WAV button: `disabled` attribute set during render and when no sounds active (not just visual opacity)
- Warning hint: amber text (`#ffcc00` on `#1e1e1e`) achieves contrast ratio ~7.5:1 (WCAG AA large text passes; verify with checker)
- Duration `<select>` is a native browser select — keyboard navigable by default
